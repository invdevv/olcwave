# OLCWave

Панель управления для [OLCRTC](https://github.com/openlibrecommunity/olcrtc), интегрированная с [Remnawave](https://docs.rw/).

OLCWave хранит шаблоны конфигураций OLCRTC (`profiles`). Когда пользователь открывает свою ссылку подписки, панель проверяет его через Remnawave, создаёт отдельную конфигурацию OLCRTC для каждого профиля, запускает необходимые Docker-контейнеры и возвращает готовую к импорту подписку.

Также панель отслеживает использование трафика пользователей и автоматически останавливает контейнеры при достижении установленного лимита.

## Возможности

* **Интеграция с Remnawave** — статус пользователя и срок действия подписки берутся напрямую из Remnawave
* **Профили** — переиспользуемые YAML-шаблоны конфигурации OLCRTC.
* **Автоматическая генерация конфигурации** — панель сама заполняет уникальные параметры для каждого пользователя (ключ шифрования, комнату Jitsi и другие значения)
* **Управление контейнерами** — запуск, остановка, перезапуск и удаление OLCRTC-контейнеров, просмотр логов в реальном времени и текущего `config.yaml`
* **Учёт трафика** — SOCKS-прокси внутри каждого контейнера считает переданные данные, backend собирает статистику по пользователям, контролирует лимиты и позволяет сбрасывать счётчики.
* **Подписки** — одна ссылка подписки для каждого пользователя, которая возвращает пакет всех доступных локаций в формате, совместимом с OLCBox


## Быстрый старт

Для запуска нужны Docker, Docker Compose и работающий инстанс Remnawave.

```bash
git clone https://github.com/invdevv/olcwave.git
cd olcwave

chmod +x install.sh
./install.sh
```

После установки откройте веб-панель и войдите, используя `ADMIN_USERNAME` и `ADMIN_PASSWORD` из файла `backend/.env`.

Или установите всё вручную:

[docs/installation.md](docs/installation.md)


## Обзор архитектуры

```text
Панель администратора             Клиент пользователя

        │                                 │
        ▼                                 ▼
Frontend (React SPA)                /sub/{uuid}
        │                                 │
        └──────────────┬──────────────────┘
                       ▼
                Backend (FastAPI)
                       │
        ┌──────────────┼─────────────────────┐
        ▼              ▼                     ▼
   PostgreSQL     Remnawave API      Docker daemon
 (пользователи,  (проверка          (запускает
  профили,        пользователя,      контейнеры
  трафик)         срока действия)    OLCRTC)
                                              │
                                              ▼
                                     Контейнеры OLCRTC
                                   olcwave-<tag>-<uuid>
```

* **Frontend** — одностраничное приложение на React + Vite. Общается с backend через HTTP с использованием JWT. В production собирается в статические файлы и обслуживается Caddy.
* **Backend** — FastAPI-приложение, которое отвечает за всю логику: аутентификацию, профили, работу с Remnawave, управление контейнерами через Docker Socket, генерацию подписок и фоновый учёт трафика.
* **PostgreSQL** — хранит локальные данные пользователей (трафик и срок действия) и шаблоны профилей.
* **Docker** — backend собирает образ `olcrtc` и запускает отдельный контейнер для каждой пары «профиль — пользователь» на Docker-хосте.
* **OLCRTC** — основной транспорт. Каждый контейнер также запускает небольшой SOCKS-прокси, который сохраняет статистику трафика в `stats.json`. Backend периодически считывает эти данные.

Подробное описание работы системы см. в [docs/architecture.md](docs/architecture.md).
